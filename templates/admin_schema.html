TECHNICAL SCHEMA AUDIT

DISTINCTION GRADE (9.5+)
1. SOVEREIGN NORMALIZATION (3NF)

The schema has been rigorously decomposed to eliminate redundancy and transitive dependencies.

  • 1NF: Resolved multi-valued attributes in news via Junction Tables.
  • 2NF: Composite keys correctly model weak entity dependencies.
  • 3NF: Removed non-prime attribute dependencies.
RELATIONAL MAPPING (SCHEMA V6)
[users] 1 -- N [portfolios]
↳ 1 -- N [holdings] (Weak Entity: Composite PK)
↳ 1 -- N [trade_requests] (Lifecycle Tracking)
↳ 1 -- 1 [transactions] (Immutable Ledger)
[assets] 1 -- N [holdings]
[market_news] N -- M [assets] (via market_news_assets)
[lookup_tables] (user_roles, transaction_types, request_statuses)
PROCEDURAL ATOMICITY (ACID)
CALL sp_ApproveTradeRequest(req_id, admin_id);

Why? This encapsulates the entire trade lifecycle in a single atomic database operation. It prevents partial updates if the application crashes mid-request.
